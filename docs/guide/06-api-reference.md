# API reference

The full API reference is generated by **typedoc** from the published `.d.ts` of every
`@buckspay/*` package, so it is mechanically faithful to the §4 contract and cannot drift
from the code.

Run `pnpm docs:api` to (re)generate it under [`docs/api/`](../api/README.md).

## Key types

- **`BuckspayClient`** — `connect` / `prepare` / `sign` / `send` / `pay` / `transfer` / `getAccountState`.
- **`AccountAdapter`** — the classic-vs-contract account-model seam.
- **`BuckspaySigner`** — *who signs* (`walletsKit`, `passkey`).
- **`Relayer`** — the facilitator client (`relay` / `buildOnboard` / `submitOnboard` / `deployContract` / `getAccountState`).
- **`RelayPayload`**, **`Receipt`** — the on-wire request / response shapes.
- **`BuckspayError`** (`BuckspayErrorCode`) — the closed error-code union.
- **`useWallet`**, **`useStellarPay`** — the React hooks.
